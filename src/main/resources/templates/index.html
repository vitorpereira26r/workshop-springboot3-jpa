This is a web app built to test and learn about web development using Java Spring Boot, React.js, and PostgreSQL.

This project simulates a workshop, where you can create users, orders, and products. There are some categories, which I only display the list of on the screen, without giving the user permission to edit, delete, or create new ones.

The users section shows the list of the users saved in the application, and it's possible to edit, delete, and create new ones. When a user is deleted, the orders associated with the user are deleted in cascade.

The orders section displays the list of orders. The orders are the most complex entity in the application, and besides creating, editing, and deleting orders, it's possible to add items to the order and pay it. Once paid, it's possible to change the order status. The orders have restrictions on exclusion, such as when a user with some orders is deleted, the orders are deleted in cascade too. When an order is deleted, the products that were in the order still exist; the only thing deleted together is the relation between products and the order, which has its table on the database.

The product section is where the products are shown as a list, and is where to create, edit, and delete the products. In the creation, beyond the name, description, and price of the product, it is necessary to pick the categories of it too, but it is not mandatory. In the editing, I didn't give permission to change the categories, only the name, description, and price. When a product is deleted, if it was in an order, the product is removed from the order.

The category section is where the application only displays the categories registered in the system. It's not possible to delete, edit, or create new ones. The categories aren't deleted when a product with this category is deleted.
